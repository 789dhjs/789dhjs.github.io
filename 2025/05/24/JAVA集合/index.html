集合概述

Java集合概览

Java集合主要有两大接口，一个都实现了Collection接口，用于接受单一元素对象，再往下则分为List,Set,Queue，三个子接口，另一个则是Map，用于接受键值对

List,Set,Queue,Map的区别

List存储的元素是有序的，可重复的

Set存储的元素是无序的，不可重复的

Queue存储的元素是按照特定顺序排列的，先进先出

Map存储的是键值对,key是无序的 不可重复的，Value是无序的，可重复的

ArrayList

底层是数组

数组

数组索引为什么从0开始

寻址公式：a[i] = baseAddress+ i * dataTypeSize

如果从1开始，那么CPU要多一道指令

a[i] = baseAddress + (i -1)* dataTypeSize

操作数组的时间复杂度(查找)

数组支持随机存储，根据下标查询元素都是O(1)

[乱序情况下]如果是根据元素值来进行查询就是O(n)

[排序情况下]根据元素值来查询就是O(logn)，二分查找

插入与删除

都是O(n)

源码

添加和扩容操作

底层原理

  • 初始化为空
  • 只有在添加数据的时候才会去扩容为10
  • 加入数据的时候先将size+1判断是不是大于当前容量大小，大于则扩容为原来的1.5倍
  • 如果有空余位置则放在size+1的位置
  • 完成后返回布尔值

ArrayList list = new ArrayList(10);这一句扩容了几次

转化

list->数组，数组->list

list转数组只需要list.toArray(new String[list.size])new String表示为字符串类型的数组

数组转list只需要array.asList(strs)

数组转list，修改数组内容，list会受影响吗？

会，这个转化不会新建对象，这两个指向的都是同一个对象

list转数组，修改list内容，数组会受影响吗？

不会，list转数组会进行一次拷贝，新建一个对象，两者没什么关系

LinkedList

底层是链表

链表

LinkedList和ArrayList的区别？

  • LinkedList底层是双向链表而ArrayList地层是动态数组
  • ArrayList按照下标查询的时间复杂度为O(1)，LinkedList不支持按照下标查询
  • 遍历：两者遍历都需要O(n)的时间复杂度
  • 插入删除：ArrayList尾部插入删除只需要O(1)复杂度，而其他插入删除则要挪动整个数组为O(n)，LinkedList头尾插入需要O(1)复杂度，而中间插入删除需要O(n)，但是比ArrayList快
  • 内存占用：ArrayList占用空间小，LinkedList要存储指针占用空间大
  • 线程安全：两者都不是线程安全的

HashMap

底层数据结构

HashMap JDK1.7和JDK1.8有什么区别

JDK1.7只有拉链法，将哈希冲突的部分存放于链表中

JDK1.8新增了红黑树，但是只有数组长度>64并且链表长度>8的时候才会转化

Put操作流程

第一次添加

  1. 第一次添加，table默认为空（判断是否为第一次的依据）
  2. 接着根据key计算索引值
  3. 判断当前位置是否为空，因为是第一次插入则必为空，然后直接插入
  4. 最后判断容量，当前容量+1是否大于负载因子*最大容量

之后添加

  1. 先判断当前table是否为空，不为空
  2. 接着根据key去计算索引值即hash(key)
  3. 查看当前索引位置的值是否为空（是否发生哈希冲突）table[i] == null?
    1. 如果发生哈希冲突，就去对比key是否存在，存在就覆盖
      1. 不存在，就判断是否使用红黑树，使用则在红黑树插入元素
        1. 未达到就先判断链表中是否存在该key，存在就直接覆盖，未存在就直接遍历链表进行尾插法
        2. 完成后先判断当前数组长度是否为64，链表长度是否为8，如果达到了就转化为红黑树
  4. 最后判断容量，如果当前容量+1>负载因子*最大容量就要进行扩容

扩容

新长度 = 旧长度*2

如果当前链表长度到达8但是数组长度未到达64，或者当前容量+1>最大容量*负载因子就会进行扩容

此时链表会进行重新分配，如果不重新分配那链表会越来越深，扩容就没有意义

JDK1.7会重新将你的链表中的 hash值%最新的长度 = 新hash值

JDK1.8：hash&老数组的长度 == 0？

= 0 为低位：保持原来的位置不变

!=0 为高位：原位置+扩容的大小

寻址算法

扰动算法：如果给的1000个数都是不够随机的相同的数，那直接进行putVal的话就会导致在同一个索引所存在的元素过多，而其他元素又没有元素，使用hash方法就可以使hash值更加均匀
