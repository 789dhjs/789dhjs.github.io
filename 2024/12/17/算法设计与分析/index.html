算法设计与分析

什么是算法

定义

算法是有规则地有限集合，是为了解决特定问题而规定的一系列操作

特征

有穷性

​ 算法应该包含优先的操作步骤

确定性

​ 算法的描述必须无歧义

有效性

​ 又称可行性，能够是可行的能够实现的

输入

​ 算法必须要有0个或以上的输入

输出

​ 算法必须要有一个或以上的输出

时间复杂度

算法中所有语句的执行时间之和

三种表示法

大O表示法

渐近上界

渐进上界O：f ( n ) = O ( g ( n ) )

（当存在正常数c和n0使得所有n>=n0有：0<= f(n) <= cg(n),eg : 3N = O(N)，即取f(n)中最高的那个阶数)

f(n) = C1n^2+C2n+C3 上界可以是n$^3$, n$^2​$

渐进下界

渐进下界O：f ( n ) = Ω ( g ( n ) )
（当存在正常数c和n0使得所有n>=n0有：0<=cg(n)<= f(n) ,取f(n)中最低的那个阶数）

f(n) = C1n^2+C2n+C3 上界可以是n$^2​$, n，1

二分查找

双指针法

L = 0,R = len(A)-1,M = (L+R)/2

空间复杂度o(1)

递归

定义

是程序直接或者间接的调用自己的一个方法

满足条件

若使递归函数有效，需要满足两个条件：

基础部分：包含return语句的部分

递归部分：自己调用自己的部分

分治

定义

​ 将一个规模为n的问题分解为k个规模较小的子问题，这些子问题互相独立且与原问题相同。递归地解这些子问题，然后将各子问题的解合并得到原问题的解。

递归与分治的关系

​ 由分治法产生的子问题往往是原问题的较小模式，反复应用分治法，可以使子问题与原问题类型一致且规模不断缩小，最终使子问题缩小到很容易求解，因此用递归求解。

用分治法解决的问题的几个特征

  • 问题的规模缩小到一定的程度可以容易地解决；
  • 问题可以分解为若干个规模较小的相同问题；
  • 利用原问题分解出的子问题的解可以合并为原问题的解；
  • 问题所分解出的各个子问题是相互独立的，即子问题之间不包含公共的子问题。如果子问题不是相互独立的，可以用动态规划法。

二叉搜索树

查找前驱节点

节点x的前驱节点使具有小于x.key的最大关键字的节点

如果x节点有左子树，x的前驱节点就是他左子树中关键字最大的节点，反之没有左子树，那么x的前驱节点可能从根节点查找到该节点经过的路劲中，为路劲中最后一个遇到的关键字小于x.key的节点。如果没有这个节点，那么x无前驱节点

左小右大，找左子树，5有左子树，所以找左子树中最大的关键字，为3

2无左子树，且没有小于它的节点故无前驱节点

15无左子树，从根开始查找，找最后一个遇到的小于15的节点 为14

查找后继节点

节点x的后继节点使具有大于x.key的最小关键字的节点

如果x节点有右子树，x的后继节点就是他左子树中关键字最大的节点，反之没有右子树，那么x的前驱节点可能从根节点查找到该节点经过的路径中，为路径中最后一个遇到的关键字大于x.key的节点。如果没有这个节点，那么x无前驱节点

17无右子树，从根开始找，遇到的最后一个大于17的节点为19

19无右子树，从根开始找，不存在大于19的节点所以19没有后继节点

图

图G = （V，E）由顶点（节点）的集V和边的集E组成，每一条边是一个顶点对（v,w）

深度优先搜索遍历图的时间复杂度为O(e+v)

广度优先搜索

单源最短路径：单一起点的最短路径问题

贝尔曼福特算法：解决一般情况下的单源最短路径

迪杰斯特拉：用的贪心(主要) 和分支限界思想解决的问题，解决非负加权图中，从起点到其他节点中的最短路径问题

3.find-set：查找某元素所在的集合
