认识Redis

特征

1.键值型(Key-Value)

2.单性能，每个命令具备原子性

3.低延迟，速度快（基于内存(*)、IO多路复用、良好的编码）

4.支持数据持久化

5.支持主从集群、分片集群

安装Redis

Redis的常见命令

5种常见数据结构

String hello world
Hash {name: “Jack”,age: 21}
List [A -> B -> C]
Set {A,B,C}
SortedSet {A: 1,B: 2,C: 3}

通用命令

KEYS：查看符合模板的所有key

通过help[command]可以查看一个命令的具体用法

DEL：删除一个指定的key 返回删除数量

EXISTS：判断key是否存在

EXPIRE：给可以设置一个有效期，到期自动删除 （-1为永久）

TTL：查看一个key的剩余有效期

不同数据结构的操作命令

String

value是字符串

  • string 普通字符串
  • int 整数类型，可以自增自减
  • float 浮点类型，可以自增自减

不管哪种格式，底层都是字节数组形式存储，只是编码方式不同，最大空间512m

set：添加或修改 get：获得 mset：批量添加 mget：批量获取 相同key会覆盖

Redis的key允许多个单词形成层级结构，多个单词用 “：” 隔开，用于解决不同业务之间相同字段的情况

​ 项目名：业务名：类型：id

Hash

vlaue是一个无序字典，类似于Java中的HashMap结构

String结构将对象序列化为JSON字符串后存储，当需要修改某个字段的时候很不方便

key Value
heima: user: 1 {name: “Jack”,age: 21}

Hash结构将每一个对象中的每个字段单独存储，可以针对单个字段做CURD

key value
field value
heima:user: 1 name Jack
age 18

hset：添加或修改 hget：获得 hmset：批量添加 hmget：批量获取 相同key会覆盖

hgetall：获取hash类型key中的所有的field和value hkeys：获取所有key中的field

hvals：获取所有key中的value

List

与Java中的LinkedList类似，可以看作是一个双向链表结构

  • 有序
  • 元素可以重复
  • 插入和删除快
  • 查询速度一般

点赞列表、评论列表等要求顺序的数据可以使用

lpush key element：像列表左侧插入一个或者多个数据 lpop key：移除返回列表左侧第一个元素，无则null

rpush key element：像列表左侧插入一个或者多个数据 rpop key：移除返回列表右侧第一个元素

lrange key star end：返回一段角标范围内的所有元素

使用List结构模拟一个栈：入口和出口在一边

使用List结构模拟一个队列：入口和出口不在一边

使用List结构模拟一个阻塞队列：入口和出口不在用一边 && 出队时采用BLOPOP或BRPOP

Set

与Java中的HashSet结构类似，可以看作为一个value为null的HashMap

  • 无序
  • 元素不可重复
  • 查找快
  • 支持交集，并集，差集等功能（实现好友列表，关注列表等功能）

sadd key member：添加1个或多个元素 srem key member：移除指定元素 scard key：返回元素个数

sismember key member：判断元素是否存在set中 smembers：获取所有元素

sinter key1 key2：求key1和key2的交集 sdiff key1 key2：求key1和key2的差集

sumion：求key1和key2的并集

SortedSed

是一个可排序的set集合

  • 可排序
  • 元素不重复
  • 查询速度快

因为这些特性，经常被用来实现排行榜这样的功能

zadd key score member：添加一个或者多个元素到sorted set，存在则覆盖

zrem key member：删除指定元素 zscore key member：获取指定元素的score值

zrank key member：获取指定元素的排名 zcard key：获取元素个数

zrange key min max：按照score排序后，获取指定范围内的元素个数

zcount key min max：统计score值在给定范围内的所有元素的个数
