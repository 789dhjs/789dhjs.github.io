1.操作系统概述

操作系统的功能(简单了解，有印象)

​ 操作系统是计算机系统中具有一定功能的软件系统。这些功能包括提供人机接口和管理计算机系统资源

管理计算机资源(有印象即可)

​ 1.处理机管理 2.存储器管理 3.设备管理 4.文件管理 5.作业管理

操作系统的类型(有印象即可)

​ 单道批处理系统、多道批处理系统、分时系统、实时系统

操作系统的特征(*重点)

1.并发

​ 两个或两个以上的事务在同一时间间隔内发生

2.共享

​ 共享是指计算机系统中的资源被多个任务共同使用

​ 1.互斥共享 :某时间内只允许一个进程访问,该进程访问期间,其他进程无法访问

​ 2.同时访问 : 某时间段内允许多个进程同时访问

3.异步

​ 进程以走走停停的不连续方式运行

4.虚拟

​ 把一个物理实体映射为多个逻辑实体

2.进程管理

顺序执行

​ 各个进程按顺序执行

程序顺序执行时的特征

​ 顺序性、封闭性、可再现性

并发执行

概念

​ 程序的并发性是指多道程序在同一时间间隔内同时发生

特征

​ 间断性、失去封闭性、不可再现性

进程的定义与特征(*)

定义

​ 进程是程序的一次执行过程,是最基本的资源分配单位

特征

​ 动态性、并发性、独立性、异步性

Bernstein条件(额外补充)

​ 若干两个程序P1 , P2 满足R(P1)$\cap$W(P2) $\cup$ R(P2)$\cap$W(P1) $\cup$ W(P1)$\cap$W(P2) = $\emptyset$ 则说眀P1P2可以并发执行

​

​ 读集 写集

S1: a = x+y R(S1) = {X,Y} W(S1) = {a}

S2 : b = Z+1 R(S2) = {Z} W(S2) = {b}

S3 : c = a-b R(S3) = {a,b} W(S3) = {c}

S4 : W = c+1 R(S4) = {c} W(S4) = {w}

S1,S2 :{x,y}$\cap${b} $\cup$ {Z}$\cap${a} $\cup$ {a}$\cap${b} =$\emptyset$ S1与S2可以并发执行

作业调度

调度算法

目标

1）周转时间短

​ 周转时间 ：从作业被提交到操作系统开始到作业终止为止这段时间间隔。对于作业来说周转时间越短越好

​ 平均周转时间：T=$\frac{1}{n}$[$\sum_{i=1}^{n}$T$_i$]

​ 带权周转时间 ：周转时间与运行时间之比 W=T/T$_s$

​ 平均带权周转时间：各个作业周转时间与作业数量之比

2）系统吞吐率高

3）处理及利用率高

调度算法

先来先服务

特点：

1.最简单的算法，可以使用队列实现，表面上很公平

2.非抢占式调度

3.有利于长作业，不利于短作业

4.有利于CPU繁忙的作业，不利于I/O繁忙的作业

影响

1.降低吞吐率

2.增加平均周转时间

内存管理

目的

方便使用者，有效利用存储资源，提高系统的工作效率

存储器管理概述

存储空间及重定位

逻辑空间与物理空间

​ 逻辑空间：用户自己的编程空间

​ 物理空间：有物理存储单元组成的空间

重定位

​ 已知逻辑地址计算物理地址

任务：

  • 重定位
  • 分配回收
  • 共享保护
  • 虚拟(扩充)

类型：

  • 静态重定位
  • 动态重定位

分区存储管理

单道程序的连续分配

​ 用于单任务单用户的系统

固定分区分配方式

​ 内碎片：一个区域分配给某个作业后，所剩下的内存不能再用，称为内碎片

​ 外碎片：一个作业加载内存时，分区的长度小于作业程序的长度，则程序无法装入，此时为外碎片

​ 在操作系统启动好时就分配好分区的个数、大小和每个分区的起始地址，在程序运行期间就不再变化

​ 缺点：产生内碎片与外碎片造成了内存的浪费

动态分区分配

分区算法

1）首次适应算法：将所有的空闲区按照地址递增，从低地址查找到高地址，直到找到一个符合要求的自由块

2）循环首次适应算法：每次从上次分配的分区起查

3）最佳适应算法：按照自由块从小到大的顺序排序(产生很多小的碎片)

4）最坏适应算法：按照自由块从大到小的顺序排序查

分区分配和回收

1）分配内存

2）回收内存(相邻合并，否则插入)

页式存储管理

页面和物理块

将进程内部的逻辑地址空间分成若干个页，将内存的物理地址空间分为若干个块，（实际上，页的大小和块的大小是相等的、且存在一一对应关系，因此可以说没有页也没有块，都是“页面”）

页表

逻辑页和物理块之间的对应关系所形成的一张映射表，也是有大小的，存储在内存中

虚地址结构

页号P 页内偏移量W

当CPU给出的虚地址长度为32位，页面的大小为4KB时，内存中共有2$^{20}$个物理块。

如果一个逻辑空间中的地址为A，页面的大小为L，则页号P和页内地址W可按下列式求得（页号从0开始）

P = (int)[A/L]

W=A%L

页内存储管理的重定位

物理地址 = 块号*页大小+页面偏移

例题

某也是存储管理的系统中，设每页大小为1KB。再某时刻，页表如表4-2所示，则逻辑地址3.4KB所对应的物理地址是多少

页号 物理块号
0 35
1 101
2 64
3 77

P = (int)(3.4KB/1KB) = 3

W = 3.4KB%1KB = 0.4KB

所以在第三页物理地址为：77*1KB+0.4KB = 77.4KB

快表

就是将要经常查询的表放入cache中，提高查询页表的速度，减少访问内存的次数在cache中的快表只需要访问一次即可。

不具有快表则需要两次访问内存

  • 第一次：访问页表
  • 第二次：得到绝对地址的内容

例题

1.如果对主存的一次存取需要1.5us，实现一次页面访问的时间为多少？

1.5*2 = 3us

2.如果系统中有快表，平均查找命中率为85%，当页表项在快表中，其查找时间忽略为0，试问其此时的存取时间为多少

1.5*0.85+0.15*1.5*2 = 1.725

多级页表

页表（也就是逻辑页面和物理块之间的对应关系的表）可能占太大的空间

每个页表项的大小为4B

一共有2$^{31-12+1 = 20}$个页

这张页表所占用的空间为4*2$^{20}​$ = 4MB

解决方法：多级页表，各级页表的大小不能超过一个页面

例题

某系统按字节编址，采用40位逻辑位，页面大小为4KB，页表项大小为4B，加入采用纯页式存储则要（）级页表，页内偏移量为（）位

答：页面大小=4KB = 2$^{12}​$B，按字节编址，因此页内偏移量为12位

页号 = 40-12 = 28

页面大小=2$^{12}$B，页表项大小=4B，则每个页可以存储2$^{12}$/4 = 2$^{10}$个页表项

需要十个2进制才能映射到2$^{10}$个页表项

  • 能够对I/O设备进行控制
  • 能够确保对设备的正确共享
  • 能够处理错误

分类：

按使用特性分类：

存储设备：外存，内存

IO设备：输入输出设备

按设备共享属性分类：

独占设备：进程互斥访问这些设备（打印机），分配独占设备时还要考虑死锁

共享设备：允许多个设备同时访问（磁盘），多个设备同时访问不影响操作正确性

按传输速率分类：

低速设备：传输速率每秒几字节到几百字节

中速设备：传输速率每秒千字节到万字节

高速设备：传输速率每秒百千字节到数十兆字节

按信息传输单位分类：

字符设备：以字节为单位

块设备：以数据块为单位

IO请求与IO应答：

各种IO模块之间的层次视图

设备控制器

数据信号线：传输数据

控制信号线：传输控制命令

状态信号线：反馈设备状态

IO控制方式DMA

当cpu需要从磁盘读入信息时，向磁盘控制器DMA发送命令，启动DMA后，CPU继续干其他事情，数据传输由DMA的控制下完成

磁盘

柱面号（第几楼）：表示要读写的磁盘的层数

磁头号（第几个走廊）：表示要读写的磁盘当前在第几个磁道

扇区号（第几号房间）：表示要读写的磁盘当前在该磁道的哪个扇区

结构：

每个扇区包含两个字段：

标识符字段：利用柱面号，磁头号，扇区号唯一的标识一个扇区

数据字段：存放512字节的数据

调度算法

先来先服务算法（FCFS）：按照请求顺序访问

最短寻找时间优先算法（SSTF）：挑选寻找时间最短的那个先执行

电梯调度算法：按照移动臂原有顺序方向一直移动，找不到后，再反方向移动

单向扫描调度算法（SCAN）：不管等待者的顺序，总是从0开始向里扫描，到最后一个柱面后再回到0重复

例题：

假定某磁盘共有200个柱面，编号为0-199，如果在为访问143号柱面的请求者服务后，当前正在为访问125号柱面的请求者服务，同时有若干请求者在等待服务，它们依次要访问的柱面号为：86、147、91、177、94、150、102、175、130。

(1) 分别用先来先服务FCFS算法，最短寻找时间优先SSTF，单向扫描算法SCAN，电梯调度算法确定实际的服务次序。

(2) 请按实际的服务次序，计算上述算法中移动臂移动的距离。

1、FCFS: 125–86–147–91–177–94–150–102–175–130。

移动距离：|125-86|+|86-147|+|147-91|+|91-177|+|177-94|+|94-150|+|150-102|+|102-175|+|175-130|=547

2、SSTF: 125–130–147–150–175–177–102–94–91–86。

移动距离：|125-130|+|130-147|+|147-150|+|150-175|+|175-177|+|177-102|+|102-94|+|94-91|+|91-86|=143

3、电梯调度算法：125–102–94–91–86–130–147–150–175–177。

移动距离：|125-102|+|102-94|+|94-91|+|91-86|+|86-130|+|130-147|+|147-150|+|150-175|+|175-177|=130

4、SCAN：125–130–147–150–175–177–0–86–91–94–102

移动的距离：|125-130|+|130-147|+|147-150|+|150-175|+|175-177|+|177-199|+|199-0|+|0-86|+|86-91|+|91-94|+|94-102|=375
